## ٢. البيانات الشخصية للطالب ✅

**حقول جديدة في جدول students + نموذج الإضافة/التعديل:**
| الحقل | النوع | التحقق |
|-------|------|--------|
| `national_id` — رقم الهوية الوطنية | TEXT | ١٠ أرقام بالضبط (regex `^[0-9]{10}$`) — اختياري |
| `birth_date` — تاريخ الميلاد | DATE | صيغة YYYY-MM-DD — اختياري |
| `parent_phone` — جوال ولي الأمر | TEXT | ٩-١٥ رقم/+ (regex `^[0-9+]{9,15}$`) — اختياري |

**تفاصيل UX:**
- حقل الهوية يرفض الأحرف غير الرقمية تلقائياً (`replace(/\D/g, '')`) ومحدَّد بـ 10 أحرف.
- رسالة خطأ حمراء أسفل الحقل إذا الإدخال غير صحيح.
- تاريخ الميلاد يعرض المعادل الهجري مباشرة.
- جوال ولي الأمر بنوع `tel` لإظهار لوحة المفاتيح الرقمية على الجوال.

**الحماية في DB:**
- `CHECK` constraints على العمودين لمنع أي إدخال غير صالح حتى لو تجاوز الواجهة.

**الملفات:**
- `app/(main)/students/page.tsx`
- `lib/db.ts` (أنواع + `getStudents` select)
- `supabase-student-personal-info-migration.sql` (جديد)

## ٣. تصدير Excel ✅

**مكتبة:** `xlsx@0.18.5` (موجودة مسبقاً في dependencies).

### أ) تصدير الطلاب (`/students`)
- زر "تصدير Excel" بجانب زر "إضافة طالب".
- يُصدِّر **القائمة المفلترة الحالية** (ليس كل الطلاب) — فتُحترم الفلاتر.
- أعمدة الملف: الاسم، الدفعة، المشرف، رقم الهوية، تاريخ الميلاد، جوال ولي
  الأمر، تاريخ الالتحاق، الحالة، الأجزاء المحفوظة، نسبة الإنجاز، آخر متابعة،
  ملاحظات.
- اسم الملف: `الطلاب_دفعة_[رقم]_[تاريخ اليوم].xlsx`.
- الورقة بصيغة RTL (`ws['!views'] = [{ RTL: true }]`).

### ب) تصدير تقارير الدفعة (`/manager/reports`)
- زر "تصدير Excel" في الهيدر.
- يصدِّر **ثلاث أوراق** في ملف واحد:
  1. **الملخص** — KPIs (إجمالي الطلاب، أجزاء، إنجاز، حضور، متابعة).
  2. **أداء المشرفين** — كل مشرف وعدد طلابه ومتوسط إنجازهم وحضورهم.
  3. **حالة المتابعة** — كل طالب وهل تمت متابعته هذا الأسبوع.
- اسم الملف: `تقرير_دفعة_[رقم]_[تاريخ].xlsx`.

**الملفات:**
- `app/(main)/students/page.tsx`
- `app/(main)/manager/reports/page.tsx`

## ٤. لوحة الدفعة — ملخص شامل ✅

**قبل:** لوحة فيها ٦ بطاقات KPI + قسم المشرفين + أربعة روابط سريعة.

**بعد:** أُضيف قسم كامل "ملخص الأقسام" بـ ٦ بطاقات، كل بطاقة تمثل قسماً من
الشريط الجانبي وتعرض لمحة عن حالته الحالية:

| البطاقة | اللمحة |
|--------|--------|
| الطلاب | العدد الكلي + حضور اليوم (نسبة ومجموع) |
| المشرفون | العدد + حضور المشرفين اليوم |
| التقارير | الإنجاز الأسبوعي + نسبة المتابعة |
| أقرب اختبار | اسم الطالب + الجزء + التاريخ والوقت |
| البرنامج القادم | اسم البرنامج + تاريخ بدايته |
| أقرب اجتماع | نوع الاجتماع + التاريخ والوقت |

كل بطاقة قابلة للنقر (عنصر `<Link>`) تفتح الصفحة المخصصة + رابط "عرض التفاصيل ←"
أسفل كل بطاقة.

**البيانات تُجلب من:** `getExams`, `getMeetings`, `getPrograms`,
`getSupervisorAttendanceForDate` بالإضافة إلى الموجود مسبقاً.

**الملفات:** `app/(main)/manager/dashboard/page.tsx`.

## ٥. البرامج التربوية — فصل بالدفعة + تحضير الطلاب ✅

### أ) التصفية بالدفعة (UI + RLS)
قديم: المشرف/المعلم يُصفَّى UI-side فقط.
جديد: **كل الأدوار المقيَّدة بدفعة** (`supervisor/teacher/batch_manager`) ترى
برامج دفعتها + البرامج العامة (`batch_id='all'`). (الحماية الحقيقية في
`supabase-batch-scope-migration.sql` من الدفعة ١.)

### ب) تحضير الطلاب داخل البرنامج
- زر بنفسجي "تحضير الطلاب" 👥 على كل بطاقة برنامج.
- ينقر يفتح مودال فيه:
  - قائمة بكل طلاب الدفعة (أو كل الطلاب إن كان `batch_id='all'`)
  - أزرار الحالات الثلاث لكل طالب: حاضر / غائب / غائب بعذر
  - أزرار تحضير جماعي + عدّادات الحالات
  - زر حفظ واحد
- جدول جديد `program_attendance` مع RLS: ceo/batch_manager/supervisor يعدلون
  طلاب دفعتهم فقط، teacher قراءة فقط.

**الملفات:**
- `app/(main)/programs/page.tsx`
- `lib/db.ts` (`DBProgramAttendance`, `getProgramAttendance`, `saveProgramAttendance`)
- `supabase-program-attendance-migration.sql` (جديد)

## ٦. الاجتماعات الدورية ✅

### إضافات:
- نوع تكرار جديد: `weekly` (أسبوعي) أو `monthly` (شهري)، أو `none` (اجتماع
  مرة واحدة).
- حقل "عدد مرات التوليد" (١-٥٢).
- عند الحفظ مع تكرار دوري: `createMeetingSeries` تولّد N اجتماعات تلقائياً
  مع تباعد ٧ أيام (أسبوعي) أو شهر ميلادي (شهري).
- كل اجتماعات السلسلة الواحدة لها `series_id` مشترك (للتمييز المستقبلي).

### التحديثات في DB:
- عمود `recurrence` (default `'none'`, check constraint).
- عمود `series_id` (nullable).
- فهرس جزئي على `series_id`.

**الملفات:**
- `app/(main)/meetings/page.tsx`
- `lib/db.ts` (`createMeetingSeries` helper)
- `supabase-meetings-recurrence-migration.sql` (جديد)
