# Timeline — Calendar Import (Phase 2)

## الغرض
تحميل تقويم أكاديمي سنوي (هجري + ميلادي + نوع اليوم) إلى قاعدة البيانات من ملف CSV/Excel.

## الأولوية: الهجري
- التواريخ تُعرض دائماً هجري كأساس، والميلادي سطر ثانوي.
- الشبكة (heatmap) مرتّبة بالشهور الهجرية (١٢ صف).
- السنة تُكتشف من الملف بناءً على `hijri_year` الأكثر تكراراً.

## الملفات

```
lib/timeline/hijri.ts              ← تحويل وتنسيق
lib/timeline/calendar-import.ts    ← Parser + Validator (CSV via papaparse, Excel via xlsx)
components/timeline/HijriDate.tsx  ← عرض تاريخ (هجري أساسي + ميلادي مساعد)
components/timeline/CalendarHeatmap.tsx  ← خريطة سنوية ١٢ × ٣٠
components/timeline/DayEditPopover.tsx   ← تعديل يوم واحد
app/(main)/timeline/calendar/page.tsx        ← قائمة التقاويم + تفعيل/حذف
app/(main)/timeline/calendar/import/page.tsx ← drop-zone + معاينة + حفظ
public/timeline-calendar-sample.csv ← عيّنة تنزيلية للمستخدم
```

## تنسيق الملف
**CSV** أو **Excel** (xlsx/xls) مع الأعمدة (تقبل عربي أو إنجليزي):

| العمود (عربي) | العمود (إنجليزي) | مطلوب؟ | النوع |
|---------------|------------------|--------|------|
| التاريخ_الهجري | hijri_date | ✅ | `YYYY-MM-DD` هجري |
| التاريخ_الميلادي | gregorian_date | ✅ | `YYYY-MM-DD` ميلادي |
| النوع | day_type | ✅ | `study` / `holiday` / `exam` / `weekend` |
| ملاحظات | notes | اختياري | نص حر |

أسماء بديلة مقبولة للنوع (case-insensitive):
- `دراسة`, `دراسي` → `study`
- `إجازة`, `عطلة` → `holiday`
- `اختبار`, `اختبارات` → `exam`
- `نهاية_الأسبوع`, `نهاية الأسبوع` → `weekend`

## خطوات الاستيراد (CEO + records_officer فقط)

1. `/timeline/calendar` → اضغط **"استيراد تقويم جديد"**.
2. اسحب الملف أو اختره (CSV/Excel).
3. الـ parser يقرأ ويعرض:
   - **أخطاء** (تمنع الحفظ): تاريخ غير صالح، نوع غير معروف، تعارض هجري↔ميلادي، تكرار.
   - **تنبيهات** (لا تمنع الحفظ): نقص أيام عن ٣٥٤/٣٥٥، يوم نهاية أسبوع مصنّف دراسة.
4. إذا لم تكن هناك أخطاء، يظهر:
   - ملخّص عدد الأيام بكل نوع.
   - جدول معاينة (أول ٤٠ سطر).
   - نموذج اسم التقويم (مقترح تلقائياً) + زر "تفعيل بعد الحفظ".
5. اضغط **حفظ** → يُنشأ `timeline_calendars` row + يُستبدل `timeline_days` للـ calendar_id.
6. إن فُعّل → كل التقاويم الأخرى تصبح `is_active=false`.

## Validation Rules

| القاعدة | المستوى |
|---------|---------|
| التاريخ الهجري بصيغة `YYYY-MM-DD` | خطأ |
| التاريخ الميلادي بصيغة `YYYY-MM-DD` | خطأ |
| `day_type` ∈ {study, holiday, exam, weekend} | خطأ |
| الهجري يطابق الميلادي بالتحويل | خطأ |
| لا تكرار للتاريخ الهجري داخل الملف | خطأ |
| الرؤوس الثلاثة الأساسية موجودة | خطأ |
| السنة الهجرية الواحدة (لا خلط) | تنبيه |
| إكمال ٣٥٤/٣٥٥ يوم | تنبيه |
| جمعة/سبت مصنّف دراسة أو اختبار | تنبيه |

## RLS
- `timeline_calendars`, `timeline_days`: قراءة للجميع، كتابة CEO + records_officer فقط.
- محاولة المشرف الكتابة تُرفض على مستوى DB.

## Test Cases

| ملف صحيح ٣٥٤ يوم لسنة ١٤٤٧ | حفظ ناجح، "تم الحفظ: العام الدراسي 1447هـ (354 يوم)" |

| ملف بهجري ١٤٤٧-٠١-٠١ لكن ميلادي ٢٠٢٠-٠١-٠١ | خطأ: "تعارض: 1447-01-01 هجري يقابل ..." |
| تبديل التقويم النشط | كل `timeline_calendars.is_active = false` ثم صف واحد `true` |
| ملف بأيام أقل من ٣٥٤ | تنبيه (لا خطأ): "ناقص X يوم من سنة YYYY" |

## Rollback
حذف الملفات أعلاه + حذف `timeline_calendars` و `timeline_days` من DB → النظام يعود لحالة Phase 1.
